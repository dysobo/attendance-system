# 热力图布局优化方案

## 问题分析

### 1. 图例位置问题
- **现状**: `.heatmap-legend` 使用 `justify-content: flex-end`,图例显示在右下角
- **问题**: 不美观,视觉上不平衡
- **需求**: 移到热力图下方居中显示

### 2. 月份标签对齐问题
- **现状**: 月份标签使用平均宽度计算 `weeks / 12 * CELL_SIZE`
- **问题**: 每个月实际周数不同,均分导致对不齐
- **需求**: 月份标签宽度与下方热力图实际宽度对齐

## 解决方案

### 二、月份标签对齐优化

#### 方案思路
1. 计算每个月实际包含的周数
2. 根据周数设置每个月份标签的固定像素宽度
3. 月份标签容器与热力图网格使用相同的左侧 padding

#### JavaScript 修改 (约第 900 行)

```javascript
// 修改前 - 简单均分
const monthWidth = computed(() => {
    const weeks = weekData.value.length;
    if (weeks === 0) return 80;
    const avgWeeksPerMonth = weeks / 12;
    return Math.max(60, Math.floor(avgWeeksPerMonth * CELL_SIZE));
});

// 修改后 - 按实际周数计算
const monthLabels = computed(() => {
    const firstWeek = weekData.value[0];
    if (!firstWeek) return months;
    const firstDay = firstWeek.days.find(d => d !== null);
    if (!firstDay) return months;
    const startMonth = new Date(firstDay.date).getMonth();
    return months.slice(startMonth);
});

// 新增 - 计算每个月实际宽度(像素)
const monthWidths = computed(() => {
    const widths = [];
    const weeks = weekData.value;
    if (weeks.length === 0) return widths;
    
    // 遍历每周,统计每个月有多少周
    const monthWeekCount = {};
    weeks.forEach(week => {
        const validDays = week.days.filter(d => d !== null);
        if (validDays.length > 0) {
            // 取该周中间的日期作为月份判断
            const midDay = validDays[Math.floor(validDays.length / 2)];
            const month = new Date(midDay.date).getMonth();
            monthWeekCount[month] = (monthWeekCount[month] || 0) + 1;
        }
    });
    
    // 按月份顺序计算宽度
    const firstWeek = weeks[0];
    const firstDay = firstWeek.days.find(d => d !== null);
    const startMonth = firstDay ? new Date(firstDay.date).getMonth() : 0;
    
    for (let i = 0; i < 12; i++) {
        const monthIndex = (startMonth + i) % 12;
        const weekCount = monthWeekCount[monthIndex] || 0;
        // 宽度 = 周数 * (格子宽度 + 间隙) = 周数 * 14px
        const width = weekCount * CELL_SIZE;
        if (width > 0) {
            widths.push(width);
        }
    }
    
    return widths;
});

// 移除旧的 monthWidth 计算
```

## 关键改进点

### 1. 使用固定像素宽度
- **格子宽度**: 12px (固定)
- **格子间距**: 2px (固定)
- **单元格尺寸**: 14px = 12px + 2px (格子+间距)
- **月份标签宽度**: 按实际周数计算,每周 14px

### 2. 月份标签精确对齐
- 统计每个月实际包含的周数
- 宽度 = 周数 × 14px
- 月份标签左侧 padding = 36px (与热力图网格对齐)

### 3. 图例居中显示
- 使用 `justify-content: center`
- 保持原有的 flex 布局结构

### 4. 响应式兼容
- 保持 `overflow-x: auto`
- 固定像素不会影响横向滚动
- 月份标签自动适应实际周数

## 效果预期

1. ✅ 图例在热力图下方居中显示,视觉更美观
2. ✅ 月份标签精确对齐下方热力图方块
3. ✅ 使用固定像素宽度,不依赖百分比
4. ✅ 保持横向滚动功能,适配窄屏
5. ✅ 代码结构清晰,易于维护

## 注意事项

1. `CELL_SIZE` 常量必须与 CSS 中的格子尺寸匹配
2. 月份标签 `padding-left` 必须与热力图网格的左侧标签宽度对应
3. 使用 `flex-shrink: 0` 防止月份标签被压缩
4. 每月周数通过取周中间日期判断,确保准确归属
