### 3. 月份宽度计算逻辑优化 ✅ (重写)

**位置**: JavaScript 671-700 行

**修改前** (原逻辑):
```javascript
// 问题: 统计的是每个月有多少天,不是多少周
weeks.forEach(week => {
    week.days.forEach(day => {
        if (day && new Date(day.date).getMonth() === monthIndex) {
            weekCount++;  // 错误: 每天都计数
        }
    });
});
widths.push(Math.max(50, weekCount * CELL_SIZE));  // 计算不准确
```

**修改后** (新逻辑):
```javascript
// 统计每个月的周数(按周的中心日期判断归属)
const monthWeekCount = {};
weeks.forEach(week => {
    const validDays = week.days.filter(d => d !== null);
    if (validDays.length > 0) {
        // 取该周中间日期判断月份
        const midDay = validDays[Math.floor(validDays.length / 2)];
        const month = new Date(midDay.date).getMonth();
        monthWeekCount[month] = (monthWeekCount[month] || 0) + 1;  // 每周计一次
    }
});

// 按月份顺序生成宽度
for (let m = 0; m < 12 - startMonth; m++) {
    const monthIndex = startMonth + m;
    const weekCount = monthWeekCount[monthIndex] || 0;
    // 宽度 = 周数 × (格子宽度 + 间隙) = 周数 × 14px
    const width = weekCount * CELL_SIZE;
    if (width > 0) {
        widths.push(width);  // 精确计算
    }
}
```

**改进点**:
- ✅ 统计每个月实际包含的**周数**,而不是天数
- ✅ 取每周中间日期判断月份归属,避免跨月周重复计数
- ✅ 宽度 = 周数 × 14px (CELL_SIZE = 12px格子 + 2px间隙)
- ✅ 使用固定像素宽度,不依赖百分比

---

## 常量定义

```javascript
const CELL_SIZE = 14; // 12px cell + 2px gap
```

**固定像素关系**:
- 格子宽度: 12px (固定)
- 格子间隙: 2px (固定)
- 单元格尺寸: 14px = 12px + 2px
- 月份标签宽度: 周数 × 14px (动态计算)
- 星期标签宽度: 32px (固定)
- 左侧对齐距离: 36px = 32px + 4px (固定)
